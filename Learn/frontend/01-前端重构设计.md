# 前端重构设计 — React 现代化方案

> 从原生 HTML/CSS/JS 迁移到 React 18 + TypeScript + Vite + shadcn/ui

---

## 1. 当前问题

| 问题 | 现状 | 影响 |
|---|---|---|
| 代码结构 | 2810 行单体 app.js，一个 class 装所有逻辑 | 改一处可能影响全局 |
| 样式系统 | 5 个 CSS 文件，颜色/间距硬编码 | 改主题要全局搜索替换 |
| 交互体验 | 无动画，消息突然出现 | 用户体验差 |
| 流式输出 | SSE 收到后整块刷新 | 无打字机效果 |
| 移动端 | 无响应式断点 | 手机端不可用 |
| 可维护性 | 无类型系统 | IDE 无智能提示 |

---

## 2. 技术选型

| 层 | 选型 | 原因 |
|---|---|---|
| 框架 | React 18 | AI 聊天生态最好 |
| 语言 | TypeScript | 类型安全，IDE 提示 |
| 构建 | Vite | 秒级 HMR，构建产物小 |
| UI 组件 | shadcn/ui | 基于 Radix UI，设计质量高，按需引入 |
| 样式 | Tailwind CSS | 设计 token 化，暗色模式内置 |
| 动画 | Framer Motion | 打字机/消息滑入/脉冲动画 |
| 流式 | Vercel AI SDK `useChat` | SSE 流式开箱即用 |
| Markdown | react-markdown + rehype-highlight | 代码高亮 |
| 安全 | DOMPurify | 防 XSS |

## 5. 动态效果清单

| 效果 | 技术 | 触发 |
|---|---|---|
| 打字机输出 | useChat + CSS cursor blink | SSE 数据到达 |
| 消息气泡滑入 | Framer Motion spring | 新消息 |
| 思考中跳动 | animate-pulse 三点 | 等待 AI 响应 |
| 侧栏展开/收起 | AnimatePresence + slide | 点击汉堡菜单 |
| 模式标签切换 | layoutId 下划线滑动 | 切换快速/流式 |
| 状态灯呼吸 | animate-pulse green/red | 端点探测更新 |
| 发送按钮变形 | SVG morph | 开始/停止流式 |
| 欢迎屏淡出 | crossfade + scale | 发送首条消息 |

## 6. API 对接

```
/api/chat          → POST {query, session_id}  → 快速模式
/api/chat_stream   → POST {query, session_id}  → SSE 流式
/api/upload        → POST FormData              → 文件上传
/ai/readyz         → GET                        → 健康检查
```

## 7. 部署

- nginx.conf 不变
- Vite build 输出到 `dist/`
- nginx `root` 指向 `dist/`
- Dockerfile 只改 `COPY dist/ /usr/share/nginx/html/`
