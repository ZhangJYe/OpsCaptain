## 这个项目是什么？

```
你（运维）说："checkoutservice CPU 告警了怎么办？"
                                  │
OpsCaption：                       ▼
  ├─ 去 Prometheus 查指标  ──→ CPU 85%，接近 limit
  ├─ 去日志平台查日志    ──→ context deadline exceeded ×12
  ├─ 去知识库查 SOP     ──→ 建议调整 HPA 阈值
  │
  └─ LLM 综合以上信息 → 生成诊断报告："根据分析，建议调整 resource limits..."
```

> **一句话：** OpsCaption 是一个用 Go 写的智能运维助手，运维人员用自然语言描述故障，系统自动查指标、查日志、查知识库，整合成带证据的诊断报告。前端用 React 构建了 AIOpsPanel 三路证据诊断面板，后端通过 Agent Contract 系统约束行为边界。

## 怎么学

### 每章的标准结构

```
📖 白话理解  → 这个模块是干什么的（用人话）
🧱 代码拆解  → 真实代码 + 逐行注释
🎯 面试问答  → 面试官会问什么，你怎么答
✅ 自测    → 3 个问题，答出来才算过关
```

### 每天学习流程

```
1. 读白话理解（5 分钟）    → 先建立直觉
2. 看架构图（5 分钟）      → 建立空间感
3. 读代码拆解（20 分钟）   → 对照真实代码
4. 练面试问答（15 分钟）   → 脱稿讲出来
5. 自测（5 分钟）         → 检验理解
```

### 核心心法

> **不要试图一下子读懂所有代码。先建地图，再走一条路，最后深入细节。**

## 面试 30 秒自我介绍

背下来，面试第一句话就说这个：

> OpsCaption 是一个用 Go 语言开发的智能运维助手。核心思路是：当运维人员遇到告警或故障时，不用在 Prometheus、日志平台、知识库之间来回切——直接用自然语言提问，系统会自动去查指标、查日志、查知识库，整合成带证据的诊断报告。
>
> 技术栈是 Go + Eino（字节的 LLM 编排框架）+ Milvus 向量数据库 + RabbitMQ 消息队列。前端用 React + TypeScript 重构了 20 个组件。我独立完成了从架构设计到编码实现的全过程，通过 Agent Contract 系统约束每个 Agent 的行为边界，目前核心链路已经跑通，正在做 RAG 基线评测。
