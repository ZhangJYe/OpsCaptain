# 验证闭环设计 — 第一优先级

> 当前统一口径（2026-05）
> - 本文中的 `Agent`、`schema gate`、`triage / reporter` 示例主要服务于历史 Harness 设计讨论。
> - 当前线上主链路请以 `Chat ReAct` 与 `AIOps Plan-Execute-Replan` 为准。

> 目标：让 Agent 系统输出从"看起来对"变成"可验证地对"

---

## 1. 问题现状

当前验证层有三个缺口：

1. **没有工具 contract test**：改 `query_log` 返回格式 → 无人知道 → Logs Agent 静默失败
2. **没有运行时 schema gate**：Agent 返回了不合规的 TaskResult → Runtime 照单全收 → Reporter 收到垃圾
3. **没有 contract enforce**：Contracts 定义了 Must/MustNot，但只是文档，Agent prompt 可能被绕过

---

## 2. 设计原则

- **增量不改旧代码**：所有新增逻辑在独立文件中，不改现有 Agent 实现
- **低误报率**：contract test 只验证关键字段（status/summary/evidence），不对完整输出做精确匹配
- **可配置**：schema gate 有开关，紧急情况可临时关闭
- **快速失败**：验证失败不阻塞整个链路，走 degraded 降级

---

## 4. 详细设计

### 4.1 Contract Validation（contracts.go 增加）

在 `Contract` 结构体上新增方法：

```go
// Validate checks whether a TaskResult satisfies this contract.
// Returns nil on success, or a descriptive error.
func (c Contract) Validate(result *protocol.TaskResult) error

// ValidateEvidence checks evidence items against evidence policy
func (c Contract) ValidateEvidence(evidence []protocol.EvidenceItem) error
```

验证逻辑：（按 Agent 不同有不同规则）

**通用规则**（所有 Agent）：
- `result.Status` 不能为空，必须是 `succeeded/failed/degraded` 之一
- `result.Summary` 不能为空
- 如果 `result.Status == degraded`，`result.DegradationReason` 必须非空
- 如果 `result.Status == failed`，`result.Error` 必须非空
- `result.Agent` 字段必须匹配 contract 的 Agent 名

**Triage 特殊规则**：
- 无 evidence（triage 不生产证据）
- Summary 必须提及 intent 或 domains

**Metrics/Logs/Knowledge 特殊规则**：
- 非 degraded 时，evidence 至少 1 条
- evidence 中每条必须有 `source_type` 和 `title`

**Reporter 特殊规则**：
- evidence 可以是 0 条（如果所有 specialist degraded）
- 涉及 degraded 时必须提及降级原因

### 4.2 Schema Gate（protocol/validate.go 新增）

运行时检查 TaskResult 结构完整性：

```go
// ValidateTaskResult performs structural validation on a TaskResult.
// This is a schema-level check (format), not a contract-level check (content).
func ValidateTaskResult(r *TaskResult) error
```

检查项：
- `task_id` 非空，符合 UUID 格式
- `agent` 非空
- `status` 是合法值
- `summary` 非空且长度 ≤ 4096
- `confidence` 在 0.0 ~ 1.0
- `evidence` 中各条目的 `source_type` 和 `title` 非空
- `error` 存在时 `code` 和 `message` 非空

### 4.3 Runtime Enforce（contracts/enforce.go 新增）

在 Runtime 中集成验证，不侵入 Agent 实现：

```go
// EnforceContract validates a TaskResult against the agent's contract.
// If validation fails, it wraps the result with Degraded status.
func EnforceContract(result *protocol.TaskResult) *protocol.TaskResult
```

调用时机：Agent 返回 TaskResult → Runtime 收集结果 → EnforceContract 检查 → 传递给 Reporter

逻辑：
1. 查找 `result.Agent` 对应的 Contract
2. 先做 Schema Gate（`ValidateTaskResult`）
3. 再做 Contract Validation（`Contract.Validate`）
4. 如果任何一步失败：
   - status → degraded
   - degradation_reason → 包含验证失败的具体原因
5. 返回（可能是修改后的）result

### 4.4 工具 Contract Tests（tools/contract_test.go 新增）

```go
// 测试 query_metrics_alerts 的输出契约
func TestQueryMetricsAlerts_Contract(t *testing.T)

// 测试 query_internal_docs 的输出契约
func TestQueryInternalDocs_Contract(t *testing.T)
```

注意：这里的 "contract test" 不是集成测试，不需要真实调用 Prometheus/Milvus。验证的是：
- 工具声明的输入参数类型是否和实际一致
- 工具的错误返回格式是否一致
- 工具在 mock 成功/失败场景下的行为是否符合 Agent 期望

### 4.5 Contract Registry 测试（contracts/contracts_test.go 新增）

```go
// 验证每个 registered contract 结构完整
func TestAllContracts_Valid(t *testing.T)

// 验证每个 agent 的 contract ID 唯一
func TestAllContracts_UniqueID(t *testing.T)

// 验证 triage contract 约束不冲突
func TestTriageContract_Consistency(t *testing.T)

// 验证 reporter contract 允许空 evidence
func TestReporterContract_EmptyEvidence(t *testing.T)
```

### 4.6 配置支持

`manifest/config/config.yaml` 新增：

```yaml
harness:
  validation:
    enabled: true              # schema gate 总开关
    contract_enforce: true     # contract enforce 开关
    strict_mode: false         # true: 验证失败拒绝任务, false: 降级为 degraded
```

---

## 5. 调用链路

```
Agent 执行完毕
  │
  ▼
TaskResult 返回
  │
  ▼
EnforceContract(result) ← contracts/enforce.go
  ├── ValidateTaskResult(result) ← 结构完整性
  ├── Contract.Validate(result)  ← 契约符合性
  └── 失败 → status=degraded, reason=具体错误
  │
  ▼
Reporter 接收（可能已降级的）result
```

---

## 6. 验收标准

运行以下命令全部通过：

```bash
go test ./internal/ai/tools/ -run Contract -v
go test ./internal/ai/agent/contracts/ -v
go test ./internal/ai/protocol/ -run Validate -v
```

验证场景：

| 场景 | 预期 |
|---|---|
| 正常 TaskResult 通过 gate | 原样返回，status 不变 |
| Summary 为空的 TaskResult | 降级为 degraded，reason 包含 "empty summary" |
| Status 为非法值的 TaskResult | 降级为 degraded，reason 包含 "invalid status" |
| Degraded 但没有 DegradationReason | 降级失败，补上 reason |
| Evidence 条目缺少 title | 降级为 degraded |
| 关闭 validation.enabled | 所有结果原样通过 |

---

## 7. 不做什么

- ❌ 不检查 evidence 内容的准确性（那是 RAG eval 的事）
- ❌ 不检查 summary 的语义质量（那是 replay test 的事）
- ❌ 不在 Agent prompt 中注入额外约束（不改 Agent 行为）
- ❌ 不修改 TaskResult 结构体定义（不新增字段）
