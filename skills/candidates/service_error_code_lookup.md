# service_error_code_lookup

## 来源

- `docs/告警处理手册.md`

## 场景

根据服务错误码快速定位常见原因。

## 当前样例

- `12000000001`：参数错误
- `12000000002`：数据库更新失败
- `12000000003`：下游接口错误
- `12000000004`：实例不存在

## 为什么适合做 skill

- 错误码到解释的映射天然结构化
- 很适合后续沉淀成 error-code lookup skill

## 更适合的运行时归属

- `knowledge`

## 下一步怎么实现

- 把错误码表结构化成可检索知识
- skill 命中后优先返回“错误码解释 + 建议排查方向”
